## Project

Nerd.ms is a Docusaurus 3.10 static site that serves as a searchable link toolbox (primarily Microsoft/Azure portals). The site is deployed via Azure Static Web Apps (workflows in `.github/workflows/`); the production URL is `https://nerd.ms`. Repo name `linkjes`; Node >=20 (required by Docusaurus 3.10).

Note: the actual project root is `nerd.ms/` (the inner directory), not the outer `Nerd.ms/`. All commands below assume `cwd = nerd.ms/`.

## Commands

- `npm start` — Docusaurus dev server
- `npm run build` — production build into `build/` (deployed by the Azure SWA workflow with `output_location: "build"`)
- `npm run serve` — serve the built site locally
- `npm run clear` — clear Docusaurus cache (`.docusaurus`, `.cache-loader`)
- `npm run build-json` — rebuild `static/commands.json` from `config/commands.csv`. The script `tasks/createCommands/build.ts` is plain JS run via `node` despite the `.ts` extension; there is no TypeScript toolchain in this repo.

There are no test, lint, or typecheck scripts.

## Architecture

### Two coexisting content systems

1. **Docs (MDX)** under `docs/`, served at the site root because `routeBasePath: '/'` is set in `docusaurus.config.js`. Sidebar is autogenerated from the folder structure (`sidebars.js`). Categories like `Microsoft Portalen`, `Toolkit`, `Personal Development` are surfaced this way.

2. **The link toolbox table** at `/start` (`src/pages/start.js`). This is the core feature: a searchable/filterable table built on `@tanstack/react-table` v8 over `static/commands.json`. Column definitions live in `src/tableHome/columns.table.js` (v8 column-def shape: `id`, `accessorKey`, `header`, `cell`, `meta.className`, optional `filterFn`); the table component (`src/components/CommandsTable/CommandsTable.js`) wires up sorting, global filter (debounced), the category select filter, and the hidden-column list (`keywords` is always hidden plus whatever callers pass via the `columnsToHide` prop). Each row links to both the destination URL and a `<command>.cmd.ms` shortcut.

### Commands data pipeline

`config/commands.csv` (semicolon-delimited; columns: `command;alias;description;keywords;category;url`) is the source of truth. Run `npm run build-json` after editing it to regenerate `static/commands.json`, which the `/start` page imports directly. The generated JSON is committed; the dev server does **not** rebuild it automatically.

When adding categories, also update the `cat-<categoryShortName>` CSS classes used by the icon cell in `columns.table.js`.

### Styling

Tailwind is wired in via a custom Docusaurus plugin inside `docusaurus.config.js` (`configurePostCss` appends `tailwindcss` + `autoprefixer`) rather than the standard Tailwind setup. Two important consequences in `tailwind.config.js`:

- `corePlugins.preflight: false` — Tailwind's reset is disabled so it doesn't fight Docusaurus's Infima styles. Don't re-enable it.
- `darkMode: ['class', '[data-theme="dark"]']` — dark mode is driven by Docusaurus's theme attribute, not a `.dark` class. Use `dark:` variants normally.

### Deploy

Two Azure Static Web Apps workflows live in `.github/workflows/` (`brave-bay-...` and `zealous-smoke-...`), both triggered on push/PR to `main`. The build is run by the `Azure/static-web-apps-deploy` action itself — not by an explicit `npm run build` step in the workflow — using `app_location: "/"` and `output_location: "build"`. Treat `main` as the deploy branch.
